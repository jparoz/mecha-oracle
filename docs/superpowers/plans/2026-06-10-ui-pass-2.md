### Task 2: Tooltip always in front via `position: fixed`

**Root cause:** `.tooltip` uses `position: absolute` inside `.card-wrap`. The action bar sits between the player sections in the DOM, and because `.card` elements have a `transform` property they form isolated stacking contexts, so tooltips from the p1 section cannot paint over the action bar. Switching to `position: fixed` with JS-calculated viewport coordinates bypasses all stacking-context constraints.

**Files:**
- Modify: `src/serve.html:85-91` (tooltip CSS)
- Modify: `src/serve.html:929-949` (mouseover handler)

- [ ] **Step 1: Change tooltip CSS to `position: fixed`**

  Lines 85-91 currently:
  ```css
  .tooltip {
    display: none; position: absolute; left: calc(100% + 4px); top: 0;
    width: 200px; background: #1e2430; border: 1px solid #4a6a9a;
    border-radius: 6px; padding: 10px; z-index: 300;
    font-size: 11px; color: #ccc; pointer-events: none;
    box-shadow: 0 4px 16px rgba(0,0,0,0.6); line-height: 1.5;
  }
  ```
  Change to:
  ```css
  .tooltip {
    display: none; position: fixed; left: 0; top: 0;
    width: 200px; background: #1e2430; border: 1px solid #4a6a9a;
    border-radius: 6px; padding: 10px; z-index: 9999;
    font-size: 11px; color: #ccc; pointer-events: none;
    box-shadow: 0 4px 16px rgba(0,0,0,0.6); line-height: 1.5;
  }
  ```

- [ ] **Step 2: Update mouseover handler to use viewport coordinates**

  Lines 929-949 currently:
  ```js

    const tooltip = wrap.querySelector('.tooltip');

      tooltip.style.left = 'calc(100% + 4px)';
      tooltip.style.right = 'auto';
    }
    if (rect.top + 250 > window.innerHeight) {
      tooltip.style.top = 'auto';
      tooltip.style.bottom = '0';
    } else {
      tooltip.style.top = '0';
      tooltip.style.bottom = 'auto';
    }
  });
  ```
  Replace with:
  ```js
  document.addEventListener('mouseover', e => {
    const wrap = e.target.closest('.card-wrap');
    if (!wrap) return;
    const tooltip = wrap.querySelector('.tooltip');
    if (!tooltip) return;
    const rect = wrap.getBoundingClientRect();
    const TW = 208; // tooltip width (200) + small buffer
    const TH = 260; // conservative max tooltip height
    let left = rect.right + 4;
    if (left + TW > window.innerWidth) left = rect.left - TW - 4;
    let top = rect.top;
    if (top + TH > window.innerHeight) top = window.innerHeight - TH - 8;
    tooltip.style.left = Math.max(8, left) + 'px';
    tooltip.style.top  = Math.max(8, top)  + 'px';
  });
  ```

  Reload. Hover over cards in the P1 zone (bottom). Tooltip should appear above the action bar, not behind it. Hover over cards near the edges; tooltip should flip to the other side rather than going off-screen.

### Task 5: Land zone-aware actionability highlighting

**Problem:** `isCardActionable` has a single branch for all `Land` cards that applies the "may I play a land this turn?" logic. This means a tapped Forest on the battlefield is always dim after playing a land this turn. Battlefield lands should be highlighted when their activated ability can fire (i.e. `can_activate: true`), and dimmed only when tapped. The land-play restriction belongs only to hand cards.

**Files:**
- Modify: `src/serve.html:387-403` (`isCardActionable`)
- Modify: `src/serve.html:628` (`cardHTML` signature)
- Modify: `src/serve.html:641` (`isCardActionable` call inside `cardHTML`)
- Modify: `src/serve.html:587-592` (six `cardHTML` call sites in `render`)
- Modify: `src/serve.html:909` (GY modal call site)

- [ ] **Step 1: Add `zone` parameter to `isCardActionable`**

  Lines 387-403 currently:
  ```js
  function isCardActionable(card, s, pid) {
    // Hand cards
    if (card.type_line.includes('Land')) {
      // Land in hand: actionable at sorcery speed when we haven't played a land yet
      return s.active_player === pid &&
        s.lands_played_this_turn === 0 &&
        (s.step === 'PreCombatMain' || s.step === 'PostCombatMain');
    }
    if (card.can_cast) return true;

    // Battlefield cards
    if (card.can_attack) return true;
    if (card.can_block) return true;
    if (card.activated_abilities && card.activated_abilities.some(a => a.can_activate)) return true;

    return false;
  }
  ```
  Replace with:
  ```js
  function isCardActionable(card, s, pid, zone) {
    // Hand lands: only playable at sorcery speed when the land-drop is available
    if (zone === 'hand' && card.type_line.includes('Land')) {
      return s.active_player === pid &&
        s.lands_played_this_turn === 0 &&
        (s.step === 'PreCombatMain' || s.step === 'PostCombatMain');
    }
    if (card.can_cast) return true;

    // Battlefield cards (including lands)
    if (card.can_attack) return true;
    if (card.can_block) return true;
    if (card.activated_abilities && card.activated_abilities.some(a => a.can_activate)) return true;

    return false;
  }
  ```
